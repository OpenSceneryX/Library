The OpenSceneryX project is a library of scenery objects for X-Plane® v8.50 and aboveNote that the Placeholder Library requires X-Plane® v8.60. It is a collaborative effort by members of the X-Plane.org community and the aim is to provide a good range of scenery components for authors to use in their scenery packages.

This is the Developer Pack, which is only to be used by scenery developers who want to use OpenSceneryX with their own projects. If you are not a scenery developer and have mistakenly downloaded this pack, you can download the full library from here.

Instructions for Scenery Developers

First, an important note regarding the licence: Because you will be referencing rather than copying objects in the library, you do not need to contact anyone or get permission from anyone to use this library with your own sceneries, whether commercial or not.

An Incredibly Quick Tutorial Using OverlayEditor

  1. Download and install the main OpenSceneryX library (from here).
  2. Once you have installed the library, 'opensceneryx' will appear in the drop-down menu on the right hand side in OverlayEditor.
  3. Select 'opensceneryx' and all the objects in the library are listed. You can select one and click the '+' sign to add it to your scenery in the normal way.

Note that some objects are found in several places in the library. A good example are the aircraft - They are organised both by type and by livery, so the same 747 might be found as:
objects/aircraft/jets/heavy/b747-400/air-canada and also objects/aircraft/livery/air_canada/b747-400.
This is to help you when building scenery, if you know you want some Air Canada aircraft you can see all types listed together, but if you know you want a 747-400 you can see all the liveries listed together. Don't worry about using the same object from different paths, X-Plane® only loads the object once.

Where multiple liveries are available for an aircraft, all appropriate liveries are also published to a generic aircraft path as well as the livery-specific path. This allows you to pick a generic aircraft and X-Plane® will automatically randomise between the available liveries. For example, if you want a random 747-400, use objects/aircraft/jets/heavy/b747-400 and you will get a different livery every time you load your scenery package. Some liveries are not included in this: One-offs, house colours and historical colours.

What You Need To Do When Distributing Your Scenery

The main problem you have is that if your users install your scenery package and your package references OpenSceneryX, then if a user doesn't have OpenSceneryX installed X-Plane® will display an 'unable to locate object definition' error on startup and then quit.

To solve this problem, naturally you will want to instruct your users to install OpenSceneryX, and the best way to do this is to link to the OpenSceneryX website. You can let your users know that your package uses OpenSceneryX by including our logo in your documentation too:

The logo can be found in the OpenSceneryX-Placeholder-${version} folder and the HTML code to insert it is: 

<a href="http://www.opensceneryx.com">
  <img src="enhancedby_opensceneryx_logo.gif" alt="Enhanced by OpenSceneryX" style="border: none;" />
</a>

However, inevitably some users will forget to download it, misunderstand or simply ignore the instructions, so to avoid X-Plane® throwing up errors, we have produced an empty version of the library, with invisible placeholder objects instead of the real objects. This is called the Placeholder Library. The idea is that you include the placeholder library inside your scenery package, so X-Plane® will not complain when it can't find OpenSceneryX. Your scenery package will load correctly, but no OpenSceneryX objects will be seen. Note that the placeholder library is very small so it will not noticeably increase the size of your scenery package.

To include the placeholder library in your package:

  1. Locate the folder titled OpenSceneryX-Placeholder-${version}, contained in this developer pack.
  2. Copy the contents of this folder into your scenery package folder.

It's as simple as that! There are only two problems you may come across: Firstly, please ensure that the version of the placeholder library you include with your scenery is the same as the version of the full library you built your scenery with, otherwise users may get 'unable to locate object definition' errors on X-Plane® startup. Secondly, if you are building a scenery package that is also a library, you will have produced your own library.txt file. In this case, simply copy-and-paste the contents of the OpenSceneryX-Placeholder-${version}/library.txt file into yours, instead of copying the whole file.

What You Need To Do if a New Version of OpenSceneryX is Released

So a new version of OpenSceneryX comes out and you want to upgrade your scenery package to use it? Well, if you don't want to use any of the new objects in the new version, you don't need to do anything at all. We guarantee that any future version of OpenSceneryX will be backward-compatible, so new releases will not break your existing scenery package. If you do want to use some of the new objects, just replace the library.txt file and opensceneryx folder inside your package with the new ones from the OpenSceneryX placeholder package and you're ready to go.

Important Notes

Note 1: Please do not distribute the full version of OpenSceneryX with your scenery package. If scenery authors do this, users will end up with multiple different versions of OpenSceneryX on their machines, and it will become a nightmare to maintain and manage. Please only distribute the placeholder library (n.b. Additionally, if you are a commercial developer, you would be breaking the licence by including OpenSceneryX in your package).


Note 2: Please do not physically copy objects out of the library into your own packages, unless you specifically ask the authors of those objects for permission. If you include individual objects without asking permission, you are breaking copyright law.


Note 3: The placeholder library will only work on X-Plane® v8.60 and above. It uses a new feature of v8.60 that allows us to reduce the priority of the placeholder objects so that they are only used if the full package is missing. The main OpenSceneryX package should work on v8.50 and above, so if you want to distribute your scenery to v8.50 users, do not include the placeholder library and ensure they install the full library.
